# Overlord 2.5.1 Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Merge upstream Overlord `2.5.1` into OverlordCyrax, preserve every Cyrax product feature, release `2.5.2`, publish to GitHub, and deploy a healthy Docker build to `root@ratbox`.

**Architecture:** Use a non-rewriting merge commit on an isolated integration branch. Capture uncommitted Cyrax product work first, merge upstream, resolve conflicts by combining upstream core behavior with Cyrax UI/routes, then validate static, unit, browser, Docker, and live health behavior before publishing.

**Tech Stack:** Go agent, Bun/TypeScript server, plain JavaScript/HTML/CSS frontend, Docker Compose, Git/GitHub, PowerShell, SSH.

## Global Constraints

- Release version is exactly `2.5.2` in server package metadata, server runtime version, and agent configuration.
- Preserve split screen, viewer, webcam multiview, wallpaper changer, trolling controls, sound settings, notification sounds, soundboard, branding, dashboard, deployment, and WinRE behavior.
- Preserve upstream privacy, virtual mode, taskbar, snapping/stopping, ping, build branding, and H.264 encoder improvements.
- Exclude `tools/vlog-transfer/**` and `docs/superpowers/plans/2026-07-13-wallpaper-transfer-localtonet-fix.md`.
- Never force-push or overwrite GitHub `main` directly.
- Keep a timestamped Docker rollback image on `root@ratbox`.
